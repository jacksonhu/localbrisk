LocalBrisk - 本地化全能 AI 智能体工作站

1. 产品概述

LocalBrisk 是一款主打“本地安全”与“极速体验”的桌面 AI 助手。它通过三层资产架构（Catalog - Schema - Asset）组织用户的本地数据、知识和智能体。

1.1 核心价值

完全本地化：数据物理隔离，模型本地推理，隐私零风险。

统一资产管理：将外部大数据引擎与本地文件系统统一资产化。

一键安装可用：极简安装包，内置轻量化推理引擎，无需配置复杂环境。

2. 视觉与交互规范 (Visual Identity)

设计语言："Clear & Float" (纯净与漂浮)。

2.1 界面美学

整体色调：纯白 (#FFFFFF) 主卡片与浅灰 (#F5F5F7) 背景的层次组合。

圆角规范：全局控件统一采用 12px 圆角。

漂浮感设计：为主体卡片添加微弱阴影（shadow-sm），营造悬浮感。

字体排版：使用 微软雅黑 或 Segoe UI，字色为深灰 (#333333)。

3. 技术架构设计 (Technical Architecture)

UI 层: Vue 3 + Tailwind CSS + Shadcn-Vue + vue-i18n。

调度层: Rust (Tauri 框架)。

核心大脑: Python (FastAPI + LangGraph + Polars)。

配置解析: 使用 PyYAML (Python) 和 yaml-rust (Rust)。

4. 目录与资产实现细节 (Deep Dive)

4.1 资产树逻辑结构 (Asset Tree Structure)

LocalBrisk 采用文件夹作为 Catalog 的物理载体，资产树根据以下逻辑进行编排：

4.1.1 树形结构定义

Catalog (Namespace)

Agent (智能体实体)

Prompts: 提示词模板、系统指令设置。

Skills:智能体的skills列表，参考标准skills定义。

Schema (逻辑库/数据集)

Models: 本地模型或者远程模型endpoint定义。

Tables: 本地 Parquet/CSV 数据或远程数据库表的映射。

Functions: 用于数据处理的自定义 SQL 或转换函数。

Volumes: 结构化/非结构化。

4.1.2 物理存储与 config.yaml 映射

Catalog 层: App_Data/Catalogs/{catalog_name}/。

包含 config.yaml：定义连接池（Connection）和 Catalog 元数据。

Agent 层: Catalog 目录下的 agents/{agent_name}/ 文件夹。

包含 agent_spec.yaml：定义提示词策略和引用的 Skills 列表。

Schema 层: Catalog 目录下的 schemas/{schema_name}/ 文件夹。

包含 schema.yaml：定义资产发现规则（是否同步远程表）。

4.1.3 YAML 配置示例

# Catalog 级别配置: config.yaml
catalog_id: "financial_research"
connection:
engine: "mysql"
host: "127.0.0.1"

# Agent 级别配置: agent_spec.yaml
agent_name: "数据分析官"
prompts:
system: "你是一个资深数据分析师..."
skills:
- "calculate_growth_rate"
- "generate_plotly_chart"

# Schema 级别配置: schema.yaml
schema_name: "q4_report"
sync_options:
auto_discover_tables: true


4.2 资产层级职能

Catalog (Data Source)：连接层。定义数据来源（MySQL, Hive, 本地）。

Agent / Schema (Logic Layer)：逻辑层。Agent 负责推理，Schema 负责数据组织。

Leaf Assets (Entity)：实体层。具体的 Skills, Tables, Prompts 等。

5. 核心功能场景

5.1 连接测试逻辑 (Connection Test)

在 Catalog 设置中点击“测试连接”，验证 config.yaml 中的 Connection 信息。成功后，所属的 Linked-Schema 会自动列出远程 Tables。

5.2 闭环生产流

用户在 Catalog 下选择一个 Agent。

Agent 调用自身绑定的 Prompts 建立人格。

Agent 访问同 Catalog 下 Schema 的 Tables 进行 Polars 高效计算。

Agent 使用 Skills 将结果转化为富文本 Note。

6. 项目路线图 (Roadmap)

Phase 1: 建立框架，集成 YAML 与 i18n。

Phase 2: 实现基于 Catalog > Agent/Schema > Assets 的递归发现机制。

Phase 3: 完成 Polars 与 LangGraph 对混合资产的统一访问。

Phase 4: 视觉美化与多语言切换。
